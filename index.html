Welcome to tracker_client’s documentation!¶

The Tracker Python API Client provides a simple Python interface for the Tracker GraphQL API, with the aim of allowing users to easily integrate data from Tracker into existing workflows and platforms.

API Reference¶

tracker_client.client¶

This module defines the Client class, used to connect to the Tracker API.

class Client(url='https://tracker.alpha.canada.ca/graphql', language='en')¶

This class represents the user’s connection to Tracker, which is established on instantiation. It allows the user to retrieve Organization and Domain objects representing organizations they are members of and domains their organization(s) control.

Parameters

  • url (str) – Tracker GraphQL endpoint, defaults to alpha endpoint.

  • language (str) – desired language to get data from Tracker in (‘en’ or ‘fr’).

Variables

gql_client (gql.Client) – gql client instance used to execute queries.

get_organizations(search='')¶

Get a list of your organizations.

Note that the optional search term is supplied as part of the GraphQL query variables and affects the API response received, rather than filtering results client-side.

Parameters

search (str) – Search term to filter results with. For example, supplying the string “canada” would return only Organizations containing “canada” in their name.

Returns

A list of your organizations.

Return type

list[Organization]

Raises

ValueError – if your organizations can’t be retrieved.

execute_query(query, params=None)¶

Executes a query on this client, with given parameters.

Intended for internal use, but if for some reason you need an unformatted response from the API you could call this.

Parameters

  • query (DocumentNode) – a gql query string that has been parsed with gql().

  • params (dict) – variables to pass along with query.

Returns

Results of executing query on API.

Return type

dict

Raises

  • TransportProtocolError – if server response is not GraphQL.

  • TransportServerError – if there is a server error.

  • GraphQLError – if query validation fails.

  • Exception – if any unhandled exception is raised within function

tracker_client.organization¶

This module defines the Domain class, which models organizations monitored by Tracker and offers methods to get data about them.

class Organization(client, name, acronym, zone, sector, country, province, city, verified, domainCount)¶

Class that represents an organization in Tracker.

Instance variables provide access to scalar fields for the organization in the GraphQL schema, while methods return JSON data for non-scalar fields. Users should not typically instantiate this class manually, instead use methods provided by Client to get Organizations.

The naming irregularity between __init__() parameters and instance variables is to match parameter names to the keys contained in the API responses. This allows easy use of dict unpacking when creating an Organization instance. Instance variable names instead adhere to Python convention.

Variables

  • client (Client) – the Client that created this object. Provides a way for Organization methods to execute queries.

  • name (str) – full name of the organization.

  • acronym (str) – acronym for the organization.

  • zone (str) – the zone the organization belongs to.

  • sector (str) – the sector the organization belongs to.

  • country (str) – country the organization resides in.

  • province (str) – province the organization resides in.

  • city (str) – city the organization resides in.

  • verified (bool) – if the organization is verified or not.

  • domainCount (int) – number of domains controlled by the organization.

  • domain_count (int) – the instance variable containing the domainCount value.

class Domain(client, domain, lastRan, dmarcPhase, selectors)¶

Class that represents a domain in Tracker.

Instance variables provide access to scalar fields for the domain in the GraphQL schema, while methods return JSON data for non-scalar fields. Users should not typically instantiate this class manually, instead use methods provided by Client to get Domains.

The naming irregularity between __init__() parameters and instance variables is to match parameter names to the keys contained in the API responses. This allows easy use of dict unpacking when creating a Domain instance. Instance variable names instead adhere to Python convention.

Variables

  • client (Client) – the Client that created this object. Provides a way for Domain methods to execute queries.

  • domain (str) – name of the domain.

  • last_ran (str) – timestamp for last scan run on the domain.
